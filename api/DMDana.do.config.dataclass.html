Usage docs: https://docs.pydantic.dev/2.7/concepts/dataclasses/

A decorator used to create a Pydantic-enhanced dataclass, similar to the standard Python dataclass, but with added validation.

This function should be used similarly to dataclasses.dataclass.

Parameters:

  • _cls ‚Äì The target dataclass.

  • init ‚Äì Included for signature compatibility with dataclasses.dataclass, and is passed through to dataclasses.dataclass when appropriate. If specified, must be set to False, as pydantic inserts its own __init__ function.

  • repr ‚Äì A boolean indicating whether to include the field in the __repr__ output.

  • eq ‚Äì Determines if a __eq__ method should be generated for the class.

  • order ‚Äì Determines if comparison magic methods should be generated, such as __lt__, but not __eq__.

  • unsafe_hash ‚Äì Determines if a __hash__ method should be included in the class, as in dataclasses.dataclass.

  • frozen ‚Äì Determines if the generated class should be a ‚Äòfrozen‚Äô dataclass, which does not allow its attributes to be modified after it has been initialized.

  • config ‚Äì The Pydantic config to use for the dataclass.

  • validate_on_init ‚Äì A deprecated parameter included for backwards compatibility; in V2, all Pydantic dataclasses are validated on init.

  • kw_only ‚Äì Determines if __init__ method parameters must be specified by keyword only. Defaults to False.

  • slots ‚Äì Determines if the generated class should be a ‚Äòslots‚Äô dataclass, which does not allow the addition of new attributes after instantiation.

Returns:

A decorator that accepts a class as its argument and returns a Pydantic dataclass.

Raises:

AssertionError ‚Äì Raised if init is not False or validate_on_init is False.
